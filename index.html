Progressive Web App

What is PWA?

Progressive Web Apps provide reliable, fast, and engaging native like experience but are delivered through the web.

Why build Apps on Web?

Web is the biggest plate form in the world. Over 5 billian devices connect to it.

Web can easily reach more individuals comparing with Apps. An average user install ZERO app per month, but could visit over 100 websites a month.

However people just pass by those sites. Mobile users spend more time on native apps instead as they are more engaging.

Combining the best of both world, we get Progressive Web App.

3 Steps to turn your website into a PWA

Web App Manifest File

A manifest is a json file that collect all the info about your applicatoin into a json file.

online manifest generator

Service Worker

A service worker is a client side programmable proxy written in Javascript btwn your app and outside world.

A service worker responds to network request and cached data as programmed.

It allows PWA to be instant loading, and able to work in low data-connection and even offline environment.

HTTPS

Progressive Web Apps requires to be served over a secure connection, unless its served from localhost for the ease of the development.

App Shell

App shell Is the static part of UI. the part of the UI remains the same regardless of the data. It ensures reliably good performance as it's loaded from the pre-cache.

App Shell architecture separates the core application infrastructure and UI from the data. So for each subsequent fetch, WPA only need to load necessary data.
